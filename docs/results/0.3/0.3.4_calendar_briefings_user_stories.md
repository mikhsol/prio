## Overview

This document contains 6 user stories for the Calendar Integration and Daily Briefings features of Prio MVP. The daily briefing is a key habit-forming feature that drives daily engagement and differentiates Prio from competitors.

---

## Context from Personas

### Alex (Overwhelmed Professional)
> "Half my day is meetings, action items get lost."

**Key needs**: Morning planning, meeting context, action item tracking

### Maya (Privacy-Conscious Creator)
> "I work all hours and forget to stop."

**Key needs**: Work boundaries, end-of-day reflection, clear "done" signal

### Jordan (Aspiring Achiever)
> "I don't know if I'm actually improving."

**Key needs**: Daily accountability, progress review, motivation

## User Stories

### CB-001: Morning Daily Briefing

**Priority**: P0  
**Persona**: Alex (Overwhelmed Professional)

**Story**:  
As a busy professional, I want a personalized morning briefing so that I start each day with clarity on what matters.

**Acceptance Criteria**:
1. [ ] Briefing available at user-configured time (default: 7:00 AM)
2. [ ] Push notification: "Good morning! Your day is ready."
3. [ ] Briefing screen shows:
   - Greeting with motivational message
   - Today's top 3 priority tasks (Q1 + top Q2)
   - Calendar overview (meetings, time blocked)
   - Goal spotlight (one goal making progress or needing attention)
   - Weather (optional, if location enabled)
4. [ ] AI-generated insight (e.g., "Heavy meeting day - protect your focus time")
5. [ ] "Start Day" button to dismiss and begin
6. [ ] Works completely offline (no weather if offline)
7. [ ] Can be viewed later from dashboard
8. [ ] Briefing regenerates if calendar/tasks change significantly

### CB-002: Calendar Read Integration

**Priority**: P0  
**Persona**: Alex (Overwhelmed Professional)

**Story**:  
As a user with many meetings, I want Prio to read my calendar so that my schedule is integrated with my task planning.

**Acceptance Criteria**:
1. [ ] Connect to device calendars (Google, Samsung, local)
2. [ ] Permission request explains data usage clearly
3. [ ] Sync calendar events for display in Prio
4. [ ] Calendar tab shows today's schedule (day view)
5. [ ] Events show: title, time, location (if any), attendees count
6. [ ] Tap event to see full details (read-only)
7. [ ] No calendar write operations in MVP (read-only)
8. [ ] Refresh manually or auto-sync every 15 minutes
9. [ ] All calendar data stays on device
10. [ ] Color-code by source calendar

**Technical Notes**:
- Use Android Calendar Provider
- Request READ_CALENDAR permission
- Store locally in Room for offline access
- Privacy-first: explain no cloud sync in permission dialog

**Dependencies**: None (foundational)

### CB-004: Meeting Notes with AI

**Priority**: P1  
**Persona**: Alex (Overwhelmed Professional)

**Story**:  
As a professional with many meetings, I want to take notes during meetings and have AI extract action items so that I never lose meeting commitments.

**Acceptance Criteria**:
1. [ ] From calendar event detail, tap "Add Notes"
2. [ ] Simple rich text editor for meeting notes
3. [ ] Voice transcription option (Android Speech API)
4. [ ] When notes saved, AI extracts action items
5. [ ] Extracted action items shown as task suggestions
6. [ ] User can accept/edit/reject each suggestion
7. [ ] Accepted action items become tasks linked to meeting
8. [ ] Notes and tasks stored locally only
9. [ ] Search notes by meeting title or content
10. [ ] View meeting history with notes attached

**Action Item Extraction Example**:
```
Meeting: Q4 Planning Review
Notes:
"We need to finalize the budget by Friday. John will 
prepare the revenue forecast. I should review the 
marketing spend and share feedback with Sarah by 
Wednesday."

AI Extracted Actions:

âœ“ [Accept] Review marketing spend (Due: Wednesday)
âœ“ [Accept] Share feedback with Sarah (Due: Wednesday)
âœ— [Reject] Prepare revenue forecast (John's task, not mine)
```

**Technical Notes**:
- Use on-device LLM for action item extraction
- Identify action items by: action verbs, first-person references, deadlines
- Link tasks to meeting via meeting_id
- Store notes in Room with FTS for search

**Dependencies**: CB-002, Task Management

### CB-005: Calendar Day View

**Priority**: P0  
**Persona**: Alex (Overwhelmed Professional)

**Story**:  
As a user, I want to see my calendar and tasks together in a day view so that I can plan my time effectively.

**Acceptance Criteria**:
1. [ ] "Today" tab shows combined calendar + task view
2. [ ] Timeline view: hours of day with events placed
3. [ ] Events show as blocks with title and duration
4. [ ] Task list shown below or beside timeline
5. [ ] Tasks with due times appear in timeline
6. [ ] Empty time blocks visible for planning
7. [ ] Tap event/task to see details
8. [ ] Swipe left/right to change days
9. [ ] Quick add task with "+" button
10. [ ] Pull to refresh calendar

**Technical Notes**:
- Use Compose Canvas or LazyColumn for timeline
- Calculate event overlaps for proper display
- Tasks without times shown in separate section
- Highlight "now" line in timeline

**Dependencies**: CB-002, Task Management

### CB-006: Smart Briefing Insights

**Priority**: P1  
**Persona**: Alex (Overwhelmed Professional)

**Story**:  
As a user, I want AI-powered insights in my briefings so that I get personalized productivity advice.

**Acceptance Criteria**:
1. [ ] Each briefing includes 1-2 AI insights
2. [ ] Insights based on: task patterns, calendar density, goal progress
3. [ ] Insight types:
   - Schedule observation ("Busy day: 5 hours of meetings")
   - Focus suggestion ("Your best focus time is mornings")
   - Goal reminder ("Your promotion goal is 3 weeks behind")
   - Celebration ("You've completed 50 tasks this month!")
   - Warning ("3 overdue tasks - consider rescheduling")
4. [ ] Insights are actionable where possible
5. [ ] Insights generated on-device using LLM
6. [ ] User can dismiss/hide unhelpful insights
7. [ ] Insights don't repeat within 7 days

**Insight Examples**:
| Context | Insight |
|---------|---------|
| 6 meetings scheduled | "Meeting-heavy day. Consider blocking 30 minutes for email catch-up." |
| No Q1 tasks | "No urgent tasks today! Great time to work on your career goal." |
| 5 overdue tasks | "You have 5 overdue tasks. Want to review and reschedule?" |
| Completed goal milestone | "ğŸ‰ You completed a milestone for 'Learn Spanish'! Keep it up!" |
| Low task completion Mon-Wed | "You tend to complete more tasks on Thursday/Friday. Plan accordingly." |

**Technical Notes**:
- Insights generated during briefing creation
- Store insight history to avoid repetition
- Use simple rules + LLM for personalization
- Track "helpful" / "dismiss" for future improvement

**Dependencies**: CB-001, Task Management, Goals
